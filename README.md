一、开发目的
客户有些产品不需要走标准核销流程，只需要对出库数量做核销。
客户做销售出库单（借条出库单）生成结算单（应收单）。在结算单中获取数量可用余额单中的未核销金额。

二、取数逻辑
1．点击借条结算单（应收单）费用明细行上的《查询可用数量》时，打开动态表单《数量可用余额单》，动态表单里面绑定的值来源于其他应付。
2．根据结算单表头的销售部门和明细页签的物料去匹配对应其他应付单的数据（销售部门和物料），然后按照部门，物料，费用项目汇总绑定到动态表单中（满足以上条件的汇总，取对应的数量）部门，物料，费用项目过滤汇总绑定到动态表单里。先数据库中取再绑定到动态表单里。
3．然后根据动态表单返回的数据，选中对应行，点击返回数据时把对应选中的行数据填充到费用明细里面（填充物料，费用项目，数量）---其他应付单和结算单部门对应。

三、回调逻辑
1.回填数量小于等于明细总数量
2.回填数量超出明细总数量
    1)在结算单保存操作后，1、2均需要将费用明细的数量反写其他应付单已用数量（新增字段），计算剩余可用数量。
    2)第2种情况下，省去动态表单回填时的校验数量超额逻辑；改为对选中返回数据中的数量依次做扣减动作，直到明细可用数量扣减为零。
    例：结算单明细数量100，动态表单选中①②两条数据数量分别为100、100
		    返回数据只记录数量为100。反写其他应付上已用数量。
	        结算单明细数量100，动态表单选中①②两条数据数量分别为90、100
		    返回数据两条，数量为90，10。将90，10反写到对应的其他应付单明细上的已用数量。
            
四.反写逻辑
1．新增反写（回调时反写，自动保存）
    (1)借条结算单中，根据明细物料匹配费用明细物料，将数量汇总到对应明细行的已结算金额，未结算金额=总金额-已结算金额。
    (2)费用明细中会记录源单（数量可用余额单）的单据编号、id和分录id。通过元数据将对应字段反写到未核销数量字段（未核销数量-费用明细数量）
2．删除反写
    (1)删除费用明细行时，在表头字段《删除反写》记录源单（数量可用余额单）的id、分录id、数量。并根据物料匹配明细，将已结算数量、未结算数量做更新。
    (2)在点击保存时，根据《删除反写》字段反写数量可用余额单；根据明细记录的借条出库单id反写出库。

注：删除单据没有反写逻辑！在以上正常操作下存在费用明细的结算单禁止删除，否则会导致数据反写存在差异，已于客户沟通得到确认答复。
